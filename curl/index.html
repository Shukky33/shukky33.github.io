Méthode HTTP

GET = lire · POST = créer · PUT = remplacer · PATCH = modifier partiellement · DELETE = supprimer · HEAD = headers uniquement (sans body)

L'URL complète incluant le protocole. Ajoutez les query params directement : ?key=val&key2=val2

Headers (-H)

Les headers transmettent des métadonnées avec la requête. Content-Type décrit le format du body. Accept précise le format attendu en retour. Authorization transporte le token d'auth.

Authentification

Basic = base64(user:pass) — non chiffré, à utiliser uniquement en HTTPS. Bearer = token OAuth2/JWT dans Authorization: Bearer <token>. API Key = clé arbitraire dans un header personnalisé.

Corps (body)

Le body transporte les données à envoyer. Utilisé avec POST, PUT, PATCH. JSON ajoute automatiquement Content-Type: application/json. Form envoie les données comme un formulaire HTML.
